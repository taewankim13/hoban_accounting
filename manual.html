호반건설 전표 이상탐지 시스템

사내 구축형 LLM 기반 실시간 전표 이상탐지 및 지능형 감사 파이프라인(FDS)의 사용자 매뉴얼입니다.
18개 탐지 룰 엔진과 Gemini Vision AI를 활용하여 전표의 이상 징후를 자동으로 분석하고 위험등급을 판정합니다.

위험등급 체계

위험등급 분류 기준

각 전표는 등록된 이상탐지 룰에 의해 분석되며, 탐지된 룰별 위험 점수의 합산으로 최종 등급이 결정됩니다.

등급 판정 기준

등급점수 기준의미조치
High합계 ≥ 0.8고위험 — 중대한 오류 또는 부정 가능성즉시 확인 및 조치
Medium0.3 ≤ 합계 < 0.8주의 — 확인이 필요한 이상 징후검토 후 판단
Low합계 < 0.3 (룰 1개+)참고 — 경미한 이상 또는 관례적 패턴모니터링
정상탐지 룰 0개정상 — 이상 징후 없음—

룰별 위험 점수

등급해당 룰점수
High 차대변 불일치(E001), 차대변 오류(E008), 중복 전표(E003), 계정 착오(E002), 고액 10억+(E004) 0.3 ~ 0.5
Medium 반복거래 이탈(E015), 역분개(E006), 해약/환불(E006-H), 고액 5억+(E004-M), 개인거래처 고액(E012), 증빙 누락(E007) 0.15 ~ 0.25
Low 주말 기표(E005), 월말 기표(E009), 단수 금액(E010), 근무시간 외(E011), 적요 미입력(E013), 다건 항목(E014) 0.05 ~ 0.1
💡
산출 예시: "차대변 불일치(0.5)" + "중복 전표(0.4)" 동시 탐지 → 합계 0.9 → High
같은 필드의 상위/하위 룰은 더 엄격한 것만 적용됩니다. (예: 10억 초과 + 5억 초과 → 10억만 적용)
화면 1

대시보드

시스템 전체 현황을 한눈에 파악할 수 있는 메인 화면입니다. 위험등급별 통계, 프로젝트별 현황, 월별 데이터 관리 기능을 제공합니다.

통계 카드

화면 상단에 5개의 통계 카드가 표시됩니다.

카드표시 내용색상
전체분석된 전체 전표 수회색
고위험High 등급 전표 수빨강
주의Medium 등급 전표 수주황
참고Low 등급 전표 수청록
정상이상 징후 없는 전표 수녹색

월 선택

통계 카드 위에 월별 필터 버튼이 표시됩니다. 특정 월을 클릭하면 해당 월의 데이터만 필터링되어 통계와 프로젝트 현황이 갱신됩니다.

프로젝트별 현황

프로젝트(현장)별 전표 건수, 고위험 건수, 총 금액을 테이블로 보여줍니다. 각 행의 검토 링크를 클릭하면 해당 프로젝트의 전표 검토 화면으로 이동합니다.

전표유형 요약 / 월별 데이터 현황

우측 사이드에는 전표유형별 건수 카드와 월별 데이터 적재 현황이 표시됩니다. 각 월의 X 버튼으로 해당 월 데이터를 삭제할 수 있습니다.

⚠
주의: 월별 데이터 삭제 시 해당 월의 전표, 분석 결과, 증빙자료가 모두 삭제됩니다. 삭제 전 확인 팝업이 표시됩니다.

데이터 업로드

월 선택 바 우측의 + 데이터 업로드 버튼으로 ERP에서 추출한 CSV 파일을 업로드합니다.

업로드 모달 열기
+ 데이터 업로드 버튼을 클릭합니다.
대상 월 선택
업로드할 데이터의 대상 월(YYYY-MM)을 선택합니다.
CSV 파일 업로드
드래그 앤 드롭 또는 파일 선택으로 CSV 파일을 업로드합니다. 업로드 후 자동으로 임포트 및 룰 분석이 시작됩니다.
진행 상황 확인
프로그레스 바로 임포트 및 분석 진행률을 확인합니다. 완료 시 자동 새로고침됩니다.
화면 2

전표 생성

수동 또는 AI 어시스턴트를 활용하여 새 전표를 생성합니다. 영수증/세금계산서 이미지를 업로드하면 AI가 자동으로 금액, 거래처, 일자를 추출하여 폼에 입력합니다.

증빙자료 업로드

화면 좌측 상단에 증빙자료 업로드 영역이 있습니다.

증빙 유형 선택
드롭다운에서 증빙 유형(세금계산서, 영수증, 거래명세서, 계약서, 공과금 고지서, 견적서, 검수확인서, 기타)을 선택합니다.
파일 업로드
파일을 선택합니다. 이미지(JPG, PNG 등)나 PDF 파일은 AI가 자동 분석하여 금액, 거래처, 일자를 추출합니다.
AI 분석 결과 확인
추출된 항목이 테이블 형태로 표시됩니다. 세금계산서의 경우 공급가액, 세액, 거래처, 작성일자 등이 자동 추출됩니다.
💡
팁: 세금계산서/영수증을 먼저 업로드하면 AI가 금액과 거래처를 자동으로 인식하여 아래 전표 폼에 반영합니다. 수동 입력의 수고를 줄일 수 있습니다.

AI 챗 어시스턴트

화면 우측 상단의 대화창에서 자연어로 전표를 생성하거나 수정할 수 있습니다.

사용 예시

입력 예시결과
자재 500만원자재비 500만원 전표 자동 생성 (부가세 포함 분개)
외주비 1000만원외주비 1,000만원 전표 생성
부가세 분리해줘기존 금액에서 부가세를 분리하여 부가세대급금 항목 추가
금액 300만원으로 변경현재 입력된 전표의 금액 수정

우측 상단의 빠른 입력 버튼(자재 500만원, 외주비 1000만원, 부가세 분리)을 클릭하면 해당 내용이 자동 입력됩니다.

전표 입력 폼

화면 하단에 전표 헤더 정보와 항목(분개) 입력 테이블이 있습니다.

헤더 정보

필드설명필수
회계연도전표의 회계연도 (기본: 현재 연도)O
전표일자전표 기표일 (기본: 오늘)O
전표유형일반전표, 매입전표, 매출전표 등O
구분 / 대분류전표의 분류 카테고리-
입력자전표를 입력하는 사원 선택O
프로젝트현장/프로젝트 검색 (자동완성)-
거래처거래처 유형 선택 + 거래처 검색 (자동완성)-
적요전표의 거래 내용 설명-

항목(분개) 입력

각 행에 차변/대변, 계정과목, 금액을 입력합니다. 계정과목 입력 시 코드 또는 과목명으로 자동완성 검색이 지원됩니다.

  • 항목 추가: + 항목 추가 버튼으로 분개 행 추가
  • 항목 삭제: 각 행의 X 버튼으로 해당 행 삭제
  • 차대 합계: 하단에 차변/대변 합계가 실시간 계산되며, 일치 여부가 표시됩니다

연결문서 첨부

전표 관련 연결문서(기안서, 계약서, 검수확인서, 품의서, 발주서, 시방서, 내역서, 기타)를 업로드합니다. 이미지/PDF 파일은 AI가 자동 분석합니다.

전표 저장

차대변 일치 확인
차변 합계와 대변 합계가 일치해야 저장할 수 있습니다.
차대변 방향 검증
계정과목의 정상 차대변 방향과 입력 방향이 다를 경우 경고 팝업이 표시됩니다.
OCR 불일치 검증
증빙자료 OCR 추출값과 입력값이 다를 경우 불일치 경고가 표시됩니다.
저장 및 분석
전표 저장 버튼을 클릭하면 전표가 저장되고 즉시 18개 룰 엔진이 자동 분석을 수행합니다.
화면 3

전표 검토

AI가 분석한 전표 목록을 다양한 필터와 검색으로 조회하고, 각 전표의 상세 분석 결과를 확인하며 승인/반려 처리를 수행하는 핵심 화면입니다.

화면 구성

좌측 사이드바
위험등급 필터(전체/고위험/주의/참고/정상)와 이상사유(룰) 필터가 세로 배치되어 있습니다.
상단 스티키 영역
검색바, 월/현장/전표유형/거래처유형 드롭다운 필터가 고정됩니다.
통계 카드
고위험/주의/참고 건수가 컴팩트 카드로 표시됩니다.
전표 목록
테이블 뷰 또는 카드 뷰로 전표를 조회합니다.

필터 및 검색

위험등급 필터 (좌측 사이드바)

좌측 사이드바의 위험등급 버튼을 클릭하면 해당 등급의 전표만 필터링됩니다. 각 버튼에 건수 배지가 표시됩니다.

이상사유 필터 (좌측 사이드바)

이상사유 버튼은 다중 선택 AND 조건으로 동작합니다. 여러 룰을 동시에 선택하면 모든 선택된 룰에 해당하는 전표만 표시됩니다.

ℹ
AND 조건: "차대변 불일치" + "고액" 두 개를 선택하면, 두 룰에 모두 해당하는 전표만 필터링됩니다.

검색

검색 대상 필드(전체, 전표번호, 일자, 전표유형, 적요, 입력자, 현장, 거래처, 계정과목)를 선택한 후 검색어를 입력합니다.

드롭다운 필터

필터설명
월특정 월의 전표만 표시
현장특정 프로젝트/현장의 전표만 표시
전표유형일반전표, 매입전표 등 유형별 필터
거래처유형법인(F), 개인(P), 부서(D), 사원(S), 기타(X), 은행(B)

테이블 뷰 / 카드 뷰

우측 상단의 뷰 전환 버튼으로 테이블과 카드 보기를 전환합니다. 선택한 뷰는 브라우저에 저장되어 다음 접속 시에도 유지됩니다.

테이블 뷰

  • 컬럼 헤더를 클릭하면 오름차순 → 내림차순 → 기본순으로 정렬이 순환됩니다
  • 컬럼: 위험등급, 전표번호, 일자, 유형, 적요, 입력자, 차변합계, 현장, 이상사유, 상태
  • 행을 클릭하면 전표 상세 모달이 열립니다

카드 뷰

3열 그리드로 전표 카드가 표시됩니다. 각 카드에 위험등급 배지, 적요, 금액, 이상사유 태그가 포함됩니다.

전표 상세 모달

전표를 클릭하면 상세 정보 모달이 열립니다. 모달은 아래 섹션으로 구성됩니다.

기본 정보
전표일자, 회계연도, 전표유형, 구분, 입력자, 현장, 사업부, 적요, 기안문서번호
AI 분석 결과
위험등급 배지, 위험 점수(%), 탐지된 이상사유 코드 태그, 탐지 사유 상세
AI 검토 제안
팝업이 열리면 Gemini AI가 각 위반 항목별로 점검 포인트, 수정 방법, 검토 절차를 자동 생성합니다. 마크다운 형식으로 표시됩니다.
첨부 증빙
영수증/세금계산서 이미지 미리보기 (있을 경우)
증빙자료 관리
증빙자료 목록 조회, 추가 업로드, AI 파싱 결과 확인, 삭제, AI 재분석
연결문서 관리
연결문서 목록 조회, 추가 업로드, AI 파싱 결과 확인, 삭제
전표 항목(분개)
차대변별 계정코드, 계정과목, 금액, 거래처 정보. 하단에 차대변 합계와 일치 여부 표시

AI 검토 제안

전표 상세 팝업이 열릴 때, 해당 전표에 이상탐지 위반이 있으면 Gemini AI가 자동으로 검토 제안을 생성합니다. 별도 버튼 클릭 없이 팝업 오픈 시점에 자동 실행됩니다.

제공 내용 (위반 항목별)

항목설명
점검 포인트해당 위반에 대해 어떤 부분을 확인해야 하는지 안내
수정 방법오류인 경우 어떻게 수정하면 되는지 구체적 방법 제시
검토 절차담당자가 밟아야 할 확인 단계를 순서대로 안내
💡
참고: AI 검토 제안은 Gemini API 키가 설정된 상태에서만 동작합니다. API 키가 없으면 "검토 제안을 생성할 수 없습니다" 메시지가 표시됩니다. 생성된 제안은 DB에 저장되어 이후 재조회 시 즉시 표시됩니다.

승인 / 반려

상태가 "제출"인 전표의 상세 모달 하단에 검토 승인과 반려 버튼이 표시됩니다.

버튼동작상태 변경
검토 승인전표를 정상으로 승인 처리제출 → 검토완료
반려전표를 반려하여 재입력 요청제출 → 반려

OCR 값 일치 점검

상단의 OCR 값 일치 점검 버튼을 클릭하면, 영수증/세금계산서를 보유한 전표들의 OCR 추출값과 전표 입력값을 비교하는 위저드가 시작됩니다.

좌측: 증빙 이미지
세금계산서/영수증 원본 이미지가 표시됩니다.
우측: 비교 테이블
작성일자, 거래처, 금액에 대해 OCR 추출값과 전표 입력값을 비교합니다. 일치/불일치 아이콘이 표시됩니다.

불일치 항목이 있으면 빨간색 경고 박스가 표시되며, AI 재분석 버튼으로 이미지를 다시 분석할 수 있습니다.

차/대변 불일치 검토

상단의 차/대변 불일치 검토 버튼을 클릭하면, 계정과목의 정상 차대변 방향과 다르게 입력된 전표를 순차적으로 검토하는 위저드가 시작됩니다.

각 전표마다 불일치 항목의 계정코드, 정상방향(녹색), 입력방향(빨간색), 금액이 테이블로 표시됩니다.

💡
참고: 역분개, 수정분개, 계정대체 등 정당한 사유로 반대 방향에 기표될 수 있습니다. 불일치가 곧 오류는 아닙니다.

Gemini Vision OCR

대화형 전표 생성 어시스턴트

전표 생성 화면의 AI 챗에서 자연어로 전표를 입력할 수 있습니다. "자재 500만원", "외주비 천만원 부가세 별도" 등의 일상적인 표현을 인식하여 계정과목, 차대변, 금액을 자동 설정합니다.

주요 기능

  • 금액 인식: "500만원", "1,000,000원", "1억" 등 다양한 금액 표현 지원
  • 계정 매핑: "자재", "외주비", "인건비" 등 일상 용어를 계정과목으로 자동 매핑
  • 부가세 처리: "부가세 포함/별도/분리" 표현 인식, 부가세대급금 자동 분리
  • 금액 수정: "금액 300만원으로 변경", "세액만 수정" 등 수정 요청 처리
  • 다건 입력: 여러 항목을 한 번에 입력 가능

AI 검토 제안

전표 검토 화면에서 전표 상세 팝업을 열면, Gemini 3.1 Pro LLM이 해당 전표의 이상탐지 결과를 분석하여 실무적인 검토 제안을 자동 생성합니다.

동작 방식

전표 팝업 오픈
전표 검토 목록에서 전표 행을 클릭하여 상세 팝업을 엽니다.
AI 검토 제안 자동 생성
위반 사항이 있는 전표인 경우, 팝업이 열리는 즉시 Gemini AI에게 검토 제안 생성을 요청합니다. 로딩 스피너가 표시됩니다.
제안 내용 표시
각 위반 항목별로 점검 포인트, 수정 방법, 검토 절차가 마크다운 형식으로 표시됩니다. 생성된 제안은 DB에 저장됩니다.

LLM 프롬프트 구성

AI 검토 제안 생성 시 다음 정보가 LLM에 전달됩니다:

  • 전표 기본 정보: 전표번호, 일자, 적요, 유형, 차변/대변 합계, 위험등급
  • 전표 라인 항목: 각 라인의 차/대변, 계정과목, 금액, 거래처
  • 감지된 위반 사항: 이상탐지 룰에 의해 감지된 모든 위반 코드 및 사유
ℹ
안내: AI 검토 제안은 참고용 가이드입니다. 최종 판단은 반드시 감사 담당자가 전표 원본과 증빙자료를 직접 확인한 후 내려야 합니다.

자연어 룰 생성

룰 관리 화면에서 자연어 설명을 구조화된 탐지 룰로 자동 변환합니다. Gemini LLM이 우선 사용되며, API 접근이 불가할 경우 로컬 패턴 매칭 엔진이 대체합니다.

지원 표현 패턴

  • 금액 조건: "N억 이상", "N만원 초과", "금액이 큰"
  • 시점 조건: "주말", "월말", "공휴일", "근무시간 외"

자주 묻는 질문 (FAQ)

Q. OCR 분석이 동작하지 않아요.

A. Gemini Vision API는 사내 네트워크에서만 접근 가능합니다. VPN 연결 상태를 확인해 주세요. 외부 네트워크에서는 OCR 없이 파일 업로드만 가능합니다.

Q. 위험등급이 실제와 다르게 판정됩니다.

A. 룰 관리 화면에서 해당 룰의 점수와 조건을 확인해 주세요. 룰 수정 후 전체 재분석을 실행하면 모든 전표에 변경된 기준이 적용됩니다.

Q. 전표를 승인/반려한 후 다시 변경할 수 있나요?

A. 현재 버전에서는 승인/반려 처리된 전표의 상태를 직접 되돌리는 기능은 제공되지 않습니다. 관리자에게 문의해 주세요.

Q. 이상사유 필터에서 여러 개를 선택하면 어떻게 되나요?

A. AND 조건으로 동작합니다. 선택한 모든 이상사유 코드가 포함된 전표만 필터링됩니다. 선택 해제는 각 버튼을 다시 클릭하거나 전체 버튼을 눌러주세요.

Q. 월별 데이터를 삭제하면 증빙자료도 삭제되나요?

A. 네, 해당 월의 전표, 분석 결과, 첨부된 증빙자료와 연결문서가 모두 삭제됩니다. 삭제 전 반드시 확인하세요.

Q. AI 챗이 응답하지 않습니다.

A. 설정에서 Gemini API 키가 올바르게 입력되었는지 확인하세요. API 키 없이는 로컬 패턴 매칭 모드로 동작하여 복잡한 요청이 처리되지 않을 수 있습니다.

Q. AI 검토 제안이 "검토 제안을 생성할 수 없습니다"라고 표시됩니다.

A. Gemini API 키가 설정되지 않았거나 네트워크 연결이 불안정한 경우 발생합니다. 우측 상단 설정(⚙️)에서 API 키를 확인하고, 사내 네트워크에 연결되어 있는지 확인하세요.

Q. 다른 브라우저에서 접속하면 AI 기능이 동작하지 않습니다.

A. API 키는 브라우저의 localStorage에 저장됩니다. 다른 브라우저나 시크릿 모드에서는 설정(⚙️)에서 API 키를 다시 입력해야 합니다.
