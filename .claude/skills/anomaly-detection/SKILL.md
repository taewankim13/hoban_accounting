# 전표 이상탐지 룰 엔진

rules.json에 정의된 룰을 전표에 적용하여 이상 여부를 판정한다.

## 아키텍처

```
rules.json (룰 설정)
    ↓
rule_engine.py (apply_rules_to_journal)
    ↓ 입력: JournalEntry + lines + _vendor_account_map + _duplicate_map
    ↓ 출력: risk_level, risk_score, error_codes, reasons, recommendations
    ↓
DB 저장 (ai_risk_level, ai_error_codes 등)
```

## 등록된 룰 (18개)

### 정합성
- **E001** 차대변 불일치 (High, 0.5) — 차변/대변 합계 차이 > 1원
- **E013** 적요 미입력 (Low, 0.05) — 적요 5자 미만
- **E014** 다건 항목 전표 (Low, 0.05) — 전표 라인 10개 이상

### 계정
- **E002** 계정과목 착오 의심 (High, 0.3) — 구분↔대분류 불일치 + 적요↔계정 불일치 13개 키워드 매핑
- **E008** 비인가 계정 조합 (High, 0.3, 비활성) — 비용↔자본 직접 상계
- **E015** 반복거래 계정과목 이탈 (Medium, 0.25) — 동일 거래처 3회+ 패턴 대비 다른 계정
- **E016** 수익/비용 차대변 오류 (High, 0.4) — 수익(41xx,43xx) 차변 / 비용(42xx,44xx,45xx) 대변 / 부채(21xx) 차변

### 금액
- **E004** 고액 전표 10억+ (High, 0.3) — max_line_amount > 10억
- **E004-M** 고액 전표 5억+ (Medium, 0.15)
- **E010** 단수 금액 (Low, 0.05) — 1만원 초과 거래 중 10원 단위 미정렬

### 중복
- **E003** 중복 전표 등록 오류 (High, 0.4) — 동일 일자+금액+계정세트+거래처

### 시점
- **E005** 주말/공휴일 기표 (Low, 0.1)
- **E009** 월말 대량 기표 (Low, 0.05) — 28일 이후
- **E011** 근무시간 외 기표 (Low, 0.05) — 7시 이전 / 22시 이후

### 유형
- **E006** 역분개 반복 (Medium, 0.2) — 적요/유형에 역분개, 취소, 반대분개
- **E006-H** 해약/환불 전표 (Medium, 0.2)

### 증빙
- **E007** 연결문서 누락 (Medium, 0.15) — 미분류 + 5천만원 초과

### 거래처
- **E012** 개인거래처 고액 (Medium, 0.15) — 개인거래처(P) + 1천만원 초과

## 룰 적용 특수 로직

### 같은 필드 중복 방지
E004(10억)과 E004-M(5억) 동시 트리거 시 점수가 높은 것만 유지한다.

### 정확한 코드 매칭
DB 필터에서 `contains('E004')`가 `E004-M`을 포함하지 않도록 LIKE 패턴 매칭한다.

### 외부 맵 주입
- `_vendor_account_map`: 거래처별 최빈 계정코드 (E015용)
- `_duplicate_map`: 동일 키 전표 그룹 (E003용)

### 위험등급 산출
- score >= 0.5 → High
- score >= 0.2 → Medium
- errors 존재 → Low
- 없음 → 정상

## 자연어 룰 생성
`parse_natural_language_rule(text)` — 금액 조건, 시점, 적요 키워드, 거래처, 항목 수 등을 자연어에서 추출하여 룰 JSON 생성.

- `rules.json` — 룰 설정 파일
- `ai_engine.py` — ERROR_CODES 정의
