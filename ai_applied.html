패턴 매칭 규칙 엔진
As-Is (기존)
감사 담당자가 수천 건의 전표를 육안으로 하나씩 검토하여 이상 여부를 판단. 전수 검사 불가능, 표본 추출 방식으로 탐지율 한계.
AI
To-Be (AI 적용)
18개 탐지 룰이 전표를 자동 전수 검사. 차대변 불일치, 중복 전표, 계정 착오, 고액 거래, 주말 기표, 증빙 누락 등 패턴을 실시간 감지.

JSON 기반 룰 정의 파일에 18개 이상의 탐지 규칙을 선언적으로 구성합니다. 각 룰은 조건식(필드·연산자·임계값)과 위험 가중치를 가지며, 룰 엔진이 전표 데이터에 순차 적용하여 위반 여부를 판정합니다. 30개 이상의 분석 필드를 지원하며, 잔액·금액·시간·계정·거래처·증빙·구조 등 7개 카테고리를 다차원으로 분석합니다. 신규 룰 추가 시 코드 변경 없이 웹 UI에서 즉시 반영할 수 있어 비개발자도 룰 관리가 가능합니다.

적용된 AI 기술

  • 패턴 매칭 엔진 — 30+ 분석 필드에 대한 조건 자동 평가
  • 거래처별 최빈 계정 분석 — 과거 거래 빈도 기반 이상 패턴 탐지
  • 중복 키 해시 맵 — 동일 일자/금액/계정/거래처 조합으로 중복 전표 감지
  • 시간대·요일 분석 — 기표 시각/요일 기반 비정상 입력 탐지
  • 수익/비용 차대변 방향 검증 — 757개 계정코드별 정상 방향 대조

탐지 룰 분류 (7개 카테고리)

  • 정합성 — 차대변 불일치, 차대변 방향 오류
  • 계정 — 계정과목 착오, 반복거래 패턴 이탈
  • 중복 — 동일 조건 중복 전표, 역분개 패턴
  • 금액 — 고액 전표, 라운드 금액, 개인거래처 고액
  • 시점 — 주말/공휴일, 월말 집중, 근무시간 외
  • 증빙 — 증빙 누락, 연결문서 미첨부
  • 내용 — 적요 미입력, 다건 분개
AI 위험등급 자동 분류
AI-based Risk Level Classification
가중 점수 알고리즘 다차원 평가
As-Is (기존)
감사 담당자의 주관적 경험에 의존하여 위험도를 판단. 담당자별 판단 기준 상이, 일관성 부족.
AI
To-Be (AI 적용)
룰 위반 유형·금액·빈도를 종합한 가중 점수 알고리즘이 4단계 위험등급을 자동 산정. 일관되고 객관적인 분류.

각 탐지 룰에는 위험 가중치(score)가 설정되어 있으며, 전표에 해당하는 룰이 위반될 때마다 가중치가 누적됩니다(최대 1.0). 최종 합산 점수를 기준으로 위험(High, ≥0.8) → 주의(Medium, 0.3~0.8) → 관심(Low, <0.3) → 정상의 4단계로 자동 분류합니다. 동일 필드에 복수 룰이 적용되면 가장 심각한 것만 유지하는 중복 방지 로직으로 과도한 위험 산정을 방지합니다.

적용된 AI 기술

  • 가중합 스코어링 — 룰별 위험도 가중치 누적 합산 (cap: 1.0)
  • 임계값 기반 분류 — 0.8/0.3 컷오프로 High/Medium/Low 자동 판정
  • 중복 제거 로직 — 동일 필드 복수 룰 중 최고 심각도만 반영
  • 프로젝트별 위험 집계 — 현장/프로젝트 단위 위험 분포 분석

업무 적용 효과

  • 위험 등급 기반으로 감사 우선순위 자동 배정
  • High 등급 전표부터 집중 검토 — 제한된 리소스의 효율적 배분
  • 대시보드에서 등급별·프로젝트별 현황 한눈에 파악
  • 객관적 기준으로 감사인 간 판단 일관성 확보

To-Be (AI 적용)
LLM이 위반 전표의 맥락을 분석하여 점검 포인트·수정 방법·검토 절차를 구체적으로 자동 생성. 전문가 수준의 가이드 제공.

두 가지 방식의 하이브리드 구조로 운영됩니다. 첫째, 18개 룰 코드별로 사전 정의된 검토 제안 템플릿이 있어, 배치 재분석 시 수천 건의 전표에 대해 LLM 호출 없이 즉시 제안을 생성합니다. 둘째, 개별 전표의 상세 검토 시 Gemini 3.1 Pro LLM이 해당 전표의 전체 정보(전표번호, 일자, 적요, 라인 항목, 위반 사유)를 분석하여 맥락에 맞는 맞춤형 검토 제안을 생성합니다. 델타 감지 캐싱으로 위반 내용이 변경되지 않은 전표는 재생성 없이 기존 제안을 유지합니다.

적용된 AI 기술

  • LLM (Gemini 3.1 Pro) — 전표 맥락 기반 맞춤형 검토 가이드 생성
  • 프롬프트 엔지니어링 — 점검·수정·절차 3단계 형식으로 응답 유도
  • 룰별 템플릿 엔진 — E001~E015 사전 정의 검토 방법 즉시 조합
  • 델타 감지 캐싱 — 위반 변경분만 재생성, 미변경분은 유지

업무 적용 효과

  • 비전문 감사인도 전문가 수준의 검토 수행 가능
  • 위반 유형별 구체적 수정 방법과 검토 절차 즉시 안내
  • 템플릿 + 캐싱으로 LLM 토큰 비용 99% 절감
  • DB 캐싱으로 동일 전표 반복 조회 시 즉시 응답

To-Be (AI 적용)

룰 관리 화면의 AI 룰 빌더에서 감사 담당자가 자연어로 탐지 규칙을 설명하면, Gemini 3.1 Pro가 시스템의 전체 룰 스키마(30+ 필드, 연산자, 심각도, 카테고리)를 참조하여 구조화된 JSON 룰을 생성합니다. AND/OR 복합 조건, 다중 필드 검사, 임계값 설정 등 복잡한 룰도 자연어만으로 생성 가능합니다. LLM 응답이 실패하거나 알 수 없는 필드를 반환하면 12개 정규식 패턴 기반 로컬 파서가 폴백으로 동작합니다.

적용된 AI 기술

  • LLM 기반 구조화 변환 — 자연어 → JSON 룰 스키마 자동 변환
  • 스키마 인식 프롬프트 — 30+ 필드/연산자/카테고리 전체 명세를 LLM에 제공
  • 필드 유효성 검증 — LLM 출력의 필드명을 known-set 대비 자동 검증
  • 정규식 로컬 폴백 — 12개 패턴(금액, 주말, 월말, 키워드 등)으로 LLM 없이 파싱

업무 적용 효과

  • 감사 담당자가 개발자 없이 직접 룰 생성
  • 복잡한 JSON 스키마 학습 불필요
  • 새로운 부정 패턴 발견 시 즉시 룰 반영 가능
  • LLM 장애 시에도 로컬 폴백으로 안정적 운영

To-Be (AI 적용)
증빙 이미지 업로드 시 Vision AI가 문서 유형을 자동 인식하고, 유형별 전용 프롬프트로 핵심 필드(금액·거래처·일자·부가세 등)를 정확 추출.

Gemini Vision API를 활용하여 4종류의 증빙 문서를 분석합니다. ① 영수증/세금계산서에서는 공급가액·부가세·합계·거래처·사업자번호·일자 9개 필드를 추출하여 전표 입력 필드에 자동 매핑합니다. ② 계약서에서는 계약명·계약 기간·계약 금액·보증금·월세 등을 추출합니다. ③ 공과금 고지서에서는 납부 기간·공급가액·부가세·당월 요금을 추출합니다. ④ 연결문서(기안서·품의서·검수확인서·도급기성 청구보고)에서는 기안일·작성자·공급가액·부가세를 추출합니다. 업로드 이미지는 자동 리사이즈(LANCZOS, 최대 2000px)되며, PDF는 PyMuPDF로 2배 해상도 PNG 변환 후 분석합니다. Gemini API 불가 시 Tesseract OCR 로컬 폴백이 작동합니다.

적용된 AI 기술

  • 멀티모달 LLM (Vision) — 이미지 입력을 직접 해석하는 Gemini Vision API
  • 문서 유형별 전용 프롬프트 — 영수증/계약서/고지서/기안서 각각 최적화
  • 이미지 전처리 — 자동 리사이즈, JPEG 품질 단계적 압축, PDF→PNG 변환
  • 건설업 계정 매핑 — 추출 키워드로 자재비·임차료·복리후생비 등 자동 매핑
  • Tesseract OCR 폴백 — 한국어+영어 OCR, 정규식 기반 금액·일자 추출

지원 문서 유형 및 추출 필드

  • 영수증/세금계산서 — 일자, 거래처, 사업자번호, 공급가, 부가세, 합계 등 9개
  • 계약서 — 계약명, 기간, 금액, 보증금, 월세 등 7개+α
  • 공과금 고지서 — 유형, 기간, 공급가, 부가세, 당월 요금 등 10개+α
  • 연결문서 — 기안일, 작성자, 부서, 현장, 공급가, 부가세 등 9개+α

Vision OCR로 추출된 증빙 데이터와 전표 정보를 룰 엔진이 자동 교차 검증. 금액 불일치·기간 미포함·거래처 불일치를 즉시 탐지.

Vision OCR이 증빙 문서에서 추출한 구조화 데이터(parsed_data)를 DB에 저장하고, 룰 엔진의 17개 증빙 관련 분석 필드가 전표 데이터와 자동 대조합니다. 텍스트 유사도 비교(토큰 기반 공통 단어 분석), 금액 정확 일치/차이 비교, 날짜 범위 포함 여부 등 다양한 연산자를 지원합니다. 계약서의 월세와 임차료 전표의 금액, 도급기성 청구보고의 금회기성과 전표 차변금액, 고지서의 당월요금과 수도광열비 전표의 대변금액 등 건설업 특화 대사 규칙이 사전 구성되어 있습니다.

적용된 AI 기술

  • Vision → 룰 파이프라인 — OCR 추출 데이터를 룰 엔진 입력으로 자동 연결
  • 토큰 기반 텍스트 유사도 — 계약서 주제/거래처와 전표 적요 간 연관성 분석
  • 날짜 범위 검증 — 전표 일자가 계약 기간 내 포함 여부 자동 확인
  • 금액 정밀 대조 — 증빙 금액(공급가·부가세·합계)과 전표 차/대변 금액 비교

건설업 특화 교차 검증 룰

  • 임차료-계약서 — 월세·보증금 금액 일치 여부
  • 수도광열비-고지서 — 당월 요금과 전표 대변 금액 대조
  • 도급기성-청구보고 — 금회기성 금액과 전표 차변 금액 대조
  • 전표-증빙 부가세 — 증빙의 부가세액과 전표 부가세 라인 대조
  • 전표-계약 연관성 — 적요와 계약 주제/거래처 공통 단어 분석

As-Is (기존)
ERP 시스템에서 계정과목 코드를 조회하고, 차변·대변 방향을 직접 판단하여 복잡한 입력 양식에 수작업 입력. 회계 지식 필수.
AI
To-Be (AI 적용)
자연어로 "자재 구매 500만원"이라고 입력하면, AI가 적정 계정과목·차대변 방향·금액을 자동 판단하여 전표 초안을 생성.

전표 생성 화면의 챗 인터페이스에서 사용자가 자연어로 거래를 설명하면, Gemini 3.1 Pro가 161개 계정과목 마스터를 참조하여 적절한 차변/대변 분개를 자동 생성합니다. 멀티턴 대화로 금액 수정("총액 1,155만원으로 변경"), 계정과목 변경, 부가세 자동 분리(공급가/세액 라인 분리) 등을 자연어 명령으로 처리합니다. LLM 사용 불가 시 10개 건설업 거래 패턴 템플릿(자재/외주/노무/장비/공사대금/선수금/출장/접대/보험/안전관리)과 정규식 기반 로컬 파서가 폴백으로 동작합니다.

적용된 AI 기술

  • 대화형 LLM — 멀티턴 대화 컨텍스트 유지, 이전 지시 참조
  • 계정과목 매핑 — 161개 마스터를 참조하여 적정 계정 자동 선택
  • 회계 분개 추론 — 거래 성격에 따른 차변/대변 방향 자동 판단
  • 부가세 자동 분리 — 합계 금액에서 공급가/세액 라인 자동 분리
  • 로컬 폴백 엔진 — 정규식 파서 + 10개 건설업 거래 패턴 템플릿

업무 적용 효과

  • 회계 비전문가도 정확한 전표 생성 가능
  • ERP 전표 입력 소요 시간 대폭 단축
  • 계정과목 코드 암기 불필요 — 자연어만으로 조작
  • 수정 작업도 대화형으로 — 재입력 없이 즉시 반영
  • LLM 장애 시에도 로컬 폴백으로 기본 기능 유지
